Input System Overview
The input system serves two distinct modes that must coexist without conflict: Environment Mode, where the player navigates the 3D world and interacts with objects, and Emulator Mode, where all relevant input is forwarded to the Amiga emulator core. Transitioning between these modes must be seamless and clearly communicated to the player.

Mode Transition
Entering Emulator Mode
The player approaches the arcade cabinet in Environment Mode. When within interaction range (approximately 1.5m), a prompt appears. Pressing the interact key (E on keyboard, A on gamepad) triggers the transition:
The player controller’s movement and camera input are disabled.
The camera smoothly interpolates to a fixed position directly in front of the screen (using a predefined camera anchor point on the cabinet actor).
The Input Mode switches to Emulator Mode. All keyboard/mouse/gamepad input is now routed to the Input Bridge.
If the emulator is not yet running, it starts. If it was already running (player stepped away previously), it resumes.
The UE cursor is confined to the game window to prevent accidental alt-tabs during mouse-driven Amiga games.
Exiting Emulator Mode
Pressing Escape while in Emulator Mode opens the pause menu. Selecting “Return to Room” reverses the transition:
Input routing reverts to Environment Mode.
The camera interpolates back to the player controller’s position.
Movement and mouselook are re-enabled.
The emulator continues running in the background (the game remains visible on the cabinet screen).

Amiga Keyboard Layout

Mouse Input
Mouse Passthrough
In Emulator Mode, mouse movement deltas (not absolute positions) are forwarded to the emulator as Amiga mouse movement. The Amiga mouse is a relative-movement device, so this maps naturally. Mouse sensitivity can be scaled via a configuration parameter. Left mouse button maps to Amiga left mouse button (fire on port 0). Right mouse button maps to Amiga right mouse button.
During Emulator Mode, the OS cursor is hidden and the mouse is confined to the application window using Unreal’s FSlateApplication cursor lock functionality. This prevents the cursor from leaving the window during play.

Analogue to Digital Conversion
The Amiga joystick is a digital 4-direction + fire device. Analogue stick input is converted with a configurable deadzone (default 0.25) and a simple threshold: any axis value beyond the deadzone maps to the corresponding digital direction. Diagonal movement is supported (both axes active simultaneously). The right stick, when mapped to mouse emulation, retains its analogue range and is scaled by a sensitivity parameter.
Input Bridge Architecture
Implementation
The Input Bridge is a C++ class (URetroScreenInputBridge) that sits between Unreal’s input system and the emulator core. It maintains the current state of all emulated input devices in a flat array of atomic integers, which the emulator thread reads during input polling callbacks. The bridge is stateless from the emulator’s perspective: it simply reflects current pressed/released state.
Data Flow
Keyboard: UE Enhanced Input Action fires → Input Bridge maps PC keycode to Amiga keycode → sets state in shared array.
Mouse buttons: UE mouse event → Input Bridge sets Amiga mouse button state.
Mouse movement: UE mouse delta → Input Bridge accumulates delta in atomic x/y accumulators → emulator reads and resets on poll.
Gamepad: UE Enhanced Input Action fires → Input Bridge maps to Amiga joystick direction + buttons → sets state in shared array.
Enhanced Input Integration
The project uses Unreal Engine’s Enhanced Input system with two Input Mapping Contexts: IMC_Environment for world navigation and IMC_Emulator for emulator control. On mode transition, the active mapping context is swapped on the player controller. The IMC_Emulator context binds every relevant key to a generic “KeyPassthrough” action that includes the key identity as a parameter, which the Input Bridge processes.
Input Configuration
User Remapping
Players can remap gamepad bindings via the pause menu. Keyboard mappings follow a fixed PC-to-Amiga translation and are not remappable in-game (the mapping is designed to feel natural). Custom keyboard overrides can be specified in the INI configuration file for advanced users. All remapping is persisted to the configuration file.
Per-Game Profiles
Different Amiga games may benefit from different gamepad mappings. The configuration system supports named input profiles that map gamepad buttons to different Amiga keys. Profiles are stored in the INI file and selected per-game. A “Default” profile covers the standard mapping described above. Users can create additional profiles for games that use unusual control schemes.
